Application Flow (Sequence Diagram)

This sequence diagram shows the flow of information for a user query, from the chatbox through the backend, model, tool server, and back to the user. It illustrates both conversational and actionable (tool-using) paths.

sequenceDiagram participant User as User (Chatbox) participant API as /api/chat (Next.js) participant Prompt as PromptEngine participant Intent as Intent Detection participant Model as Ollama Model participant MCP as MCPClient participant MCPServer as MCP Server participant Tools as Tool Definitions participant Data as Data Store User->>API: Send message API->>Prompt: Build prompt Prompt->>Intent: Detect intent API->>Intent: Check if conversational alt Conversational API->>Model: Send prompt (no tools) Model->>API: Respond (text) API->>User: Show response else Actionable API->>MCP: Get available tools API->>Model: Send prompt + tools Model->>API: Tool call request API->>MCP: Call tool MCP->>MCPServer: Execute tool MCPServer->>Tools: Use tool definition MCPServer->>Data: Read/write data MCPServer->>MCP: Return result MCP->>API: Return tool result API->>Model: Send tool result Model->>API: Final response API->>User: Show response end
